# my_robot_py_sim

ROS 2 simulation package for a mobile manipulator in Gazebo and RViz2.

## Features

- Mobile manipulator URDF with base, wheels, torso, head, sensor mount, LiDAR housing, and fixed arms
- Gazebo world with door frame, boundary walls, and mixed obstacles
- Keyboard velocity control through `/cmd_vel`
- Odometry and `odom -> base_footprint` TF generated from Gazebo model pose
- RViz2 visualization with robot model, TF, grid, base footprint, and safety shell
- Head-mounted 3D GPU LiDAR bridged to ROS on `/lidar/points`
- 2D projected scan on `/scan` and online SLAM map on `/map`

## Planning Shapes

This package keeps two robot shapes separate:

- Base footprint: a 2D rectangle in `base_footprint`, published on
  `/base_footprint_marker` for RViz inspection and recorded in
  `config/nav2_footprint.yaml` for future Navigation2 costmaps.
- Safety shell: a full-body 3D shell published on `/safety_shell_array`.
  This includes the base, wheels, torso, head, sensor mount, LiDAR housing,
  arms, and hands, and is meant for future 3D clearance checks.

The footprint answers whether the mobile base can pass on the ground plane.
The safety shell answers whether the whole fixed-pose mobile manipulator can
pass through the surrounding 3D space.

## Sensor Topics

- `/lidar/points`: 3D point cloud from the head-mounted Gazebo GPU LiDAR.
- `/scan`: 2D LaserScan projected from `/lidar/points`.
- `/map`: 2D occupancy grid generated by `slam_toolbox`.

`/odom` is generated from Gazebo's actual `mobile_manipulator` model pose, so
RViz and SLAM follow the simulated robot pose when the robot is blocked.

Navigation2 and automatic path planning are intentionally not started yet.

## Run

```bash
cd ~/ros2_ws
source install/setup.bash
ros2 launch my_robot_py_sim sim_with_rviz.launch.py
```

Localization with a saved map:

```bash
cd ~/ros2_ws
source install/setup.bash
ros2 launch my_robot_py_sim localization_with_rviz.launch.py map:=/home/jensen/ros2_ws/my_first_map.yaml
```

Keyboard control:

```bash
cd ~/ros2_ws
source install/setup.bash
export ROS_DOMAIN_ID=23
export ROS_LOCALHOST_ONLY=1
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```
